# 특허 검토 웹앱 — 디자인 목업

변리사 의뢰 사내 검토 도구의 정적 디자인 프로토타입. 1280px 가로 기준. Wanted Design System 위에 더 중립적인 회색·검정 스킨을 덧입혀 "작업 도구" 느낌으로 톤다운.

## 화면

| 파일 | 내용 |
|---|---|
| `index.html` | 특허 목록 — 빽빽한 표, 좌측 분류·상태 레일, 상단 진행률, 필터 칩, 정렬, CSV 내보내기 |
| `detail-v1.html` | 상세 V1 — **우측 고정 챗봇** (3분할 풀 레이아웃, 데스크탑 기본) |
| `detail-v2.html` | 상세 V2 — **하단 드로어 챗봇** (필요할 때만 토글, 본문 가독성 우선) |
| `detail-v3.html` | 상세 V3 — **전체화면 모달 챗봇** (집중 모드, 좌측에 특허 컨텍스트 카드 + 우측에 풀 채팅) |

상단바의 v1/v2/v3 토글로 즉시 변형 비교 가능. 좌측 레일에서 다른 특허 클릭 시 동일 변형 유지.

## 어떻게 살펴보나

1. `index.html` 열기 → 표에서 아무 줄이나 클릭하면 상세 페이지로 (기본 v1)
2. 상단 우측 v1/v2/v3로 변형 전환
3. 좌측 레일에서 다른 특허로 이동 — 4건의 특허 (EUV 펠리클, Intel HfO2, LG 리튬황, BMS PD-L1)에는 풍부한 가상 요약과 샘플 챗봇 대화가 포함되어 있음. 나머지는 자동 생성 플레이스홀더

## 컴포넌트 구조

- `tokens.css` — Wanted 시스템 위에 회색 중심으로 변수 재정의 (`--pr-bg`, `--pr-fg`, ...)
- `list.css`, `detail.css` — 페이지별 추가 스타일
- `data.js` — 20건 가상 특허 (반도체·전자·바이오, 한국·미국 혼합)
- `summaries.js` — 마크다운 형식 가상 AI 요약 + placeholder 생성기
- `markdown.jsx` — 작은 마크다운 → JSX 렌더러 (H2/H3, bold, ol/ul, p, code 지원)
- `icons.jsx` — Lucide 스타일 인라인 아이콘
- `TopBar.jsx`, `DetailRail.jsx`, `SummaryPanel.jsx`, `ChatPanel.jsx`, `PatentList.jsx`, `DetailPage.jsx`

## 변형 비교 (3가지)

**V1 — 우측 고정 (260 + 가변 + 380)**
- 장점: 검토자가 요약과 챗봇을 동시에 보면서 작업 — 끊김 없는 컨텍스트 스위칭
- 단점: 본문 가독 영역이 좁아짐 (~720px max-width 유지 필요). 1366px 이하에서 답답할 수 있음

**V2 — 하단 드로어 (FAB 트리거)**
- 장점: 본문 풀 가독. 챗봇은 필요할 때만 호출, 항상 위에 떠 있음
- 단점: 본문 일부가 가려질 수 있고, 한 번에 본문+챗봇 모두 보긴 어려움

**V3 — 전체화면 모달 (좌 컨텍스트 + 우 채팅, 920×720)**
- 장점: 챗봇에 집중. 좌측 컨텍스트 카드로 특허 핵심 정보가 항상 보임
- 단점: 모달이 열리면 본문은 가려짐 → "본문 다시 보고 질문하기" 흐름이 끊김
